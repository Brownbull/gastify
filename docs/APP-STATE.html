Biggest Gaps vs Legacy

1. No Dashboard / Home Analytics high

BoletApp's home screen was a rich treemap showing "Este Mes" (This Month) spending by category, with recent transactions expandable below. Gastify's home is a simple hub with navigation links. The dashboard was the first thing users saw and set the tone for the app as an analytics product, not just a scanner.

2. No Charts / Trends View high

BoletApp had 5 chart types (Treemap, Donut, Sankey, Bump, Radar) with multi-level drill-down (store groups → store categories → item groups → items) and temporal navigation (year/quarter/month/week). Gastify's insights are text-only monthly rollups — functional but visually flat.

  • Treemap: category spending at a glance
  • Donut: store/item distribution
  • Sankey: store → item category flow
  • Bump: time-series category trends
  • Radar: comparative category analysis

3. No Settings Screen high

BoletApp had a comprehensive settings view with 9 sub-sections: profile, preferences (language, currency, date format, themes), scan defaults, subscription/credits, learned data management, groups, app settings, and account (export, wipe, sign-out). Gastify has zero settings UI despite having backend APIs for profile rectification and data portability.

4. No Items / Products View medium

BoletApp had a dedicated "Productos" screen showing all scanned items across transactions, with the same filter/search capabilities as the transaction list. Gastify shows item-level data only within a transaction's detail view — there's no way to search "how much did I spend on chicken across all transactions."

5. No Reports medium

Weekly/monthly spending summaries with donut chart visualizations. Not just raw data — formatted cards with insights. Gastify's insights endpoint covers some of this data but the presentation is basic.

6. No Theme Switcher medium

BoletApp ran 3 color themes (Normal, Professional, Mono) × light/dark mode = 6 visual variants. The theme system was designed and stress-tested across mockups. Gastify has no runtime theme control — there's one appearance. The design tokens were specced (in the legacy mockup phase) but never wired to a runtime toggle.

7. No Batch Scan low

BoletApp let users capture multiple receipts in a batch, then review each one before saving. Gastify processes one scan at a time. Batch scanning was a power-user feature — most users scanned one receipt at a time.

8. No Notification Center low

BoletApp had an in-app alerts view (read/unread, mark read, delete). Gastify has push token registration on mobile but no UI to view or manage notifications. Less critical until push notifications are actively being sent.

9. No Category/Merchant Management low

BoletApp's "Datos Aprendidos" let users view, edit, and delete the system's learned category/merchant/item/subcategory mappings. Gastify's categorization is fully AI-driven with no user override UI. Less critical if AI accuracy is high, but reduces user control.

What Gastify Added

Statement Reconciliation major

Full bank statement pipeline: PDF upload, AI extraction, reconciliation engine matching statement lines to receipt transactions. Buckets (matched, statement-only, app-only, ambiguous) with coverage metrics. BoletApp only had a placeholder.

Privacy & Compliance (4 jurisdictions) major

Law 21.719 (Chile) + GDPR + PIPEDA + CCPA/CPRA compliance: consent management with audit trail, data subject request APIs (access, rectification, erasure, portability), processing register. Not in BoletApp.

PostgreSQL RLS + Non-Superuser Roles infra

Defense-in-depth: runtime connects as a least-privilege role; RLS policies enforce tenant isolation at the database. Startup guard refuses to boot if the role can bypass RLS. BoletApp used Firebase (no SQL, no RLS).

Scan Review Signals & Math Gate

Extraction quality surfaced to users: math reconciliation, confidence scoring, review-level classification. BoletApp trusted extraction output silently.

Boleta Barcode Shortcut

Chilean boletas with SII TED barcodes are parsed without LLM calls (0 tokens, 0 cost). Falls through to vision extraction if no barcode detected.
