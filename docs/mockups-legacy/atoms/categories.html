Categories — L1 Rubros

12 store-category groups (V4 taxonomy, Story 17-2). Each group has its own foreground/background/border color triplet defined per theme × mode = 6 variants. Background, foreground, and border come from --cat-<key>-{bg|fg|border} tokens in desktop-shell.css; emojis come from STORE_GROUP_INFO. Switch the Tweaks panel theme to see the swatches recolor.

Source: categoryColors/groupColors.ts (STORE_GROUP_COLORS) + categoryColors/groups.ts (STORE_GROUP_INFO emoji + name map).

Token usage pattern

Each L1 Rubro exposes three tokens, automatically themed:

  • --cat-<key>-fg — text and icon foreground (4.5:1+ contrast against bg)
  • --cat-<key>-bg — chip/pill/badge surface; flips dark in dark themes
  • --cat-<key>-border — accent edge for emphasis (group headers, active filters)

Apply via .{atom}--cat-<key> selectors (defined in atoms.css) for the 6 most common categories. Less-common categories use inline style="background: var(--cat-<key>-bg); ..." — see cat-card swatches above.

Used by molecules (one level up)

Populated during Phase L2. Anticipated consumers: card-transaction (category badge in transaction row), filter-strip (category chip filters), insight-card (group-colored accent), report-row (group breakdown bars).

Coverage gaps (deferred)

Phase L1 ships the 12 L1 Rubros only. Follow-up phases:
  • L3 Familias (9 item-category groups) — food-fresh / food-packaged / food-prepared / salud-cuidado / hogar / productos-generales / servicios-cargos / vicios / otros-item. Same token pattern, different prefix (--itemcat-<key>-*).
  • L2 Giros (44 store categories) + L4 Categorias (42 item categories) — leaf-level colors. Inherit from their group's color triplet by default (per frontend convention); explicit overrides only where the design diverges.
  • SVG icon set — replace emojis with custom SVGs once a designer commits to the visual style.
