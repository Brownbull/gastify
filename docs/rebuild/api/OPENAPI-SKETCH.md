# OpenAPI Contract Sketch â€” A.9

Phase A prereq A.9. Enumerates every endpoint the frontend needs, grouped by resource. Cross-references A.10 schema primitives and SCOPE REQs. This sketch becomes the source for the actual `openapi.yaml` generated by the FastAPI backend (A.11).

## Design Principles

1. **RESTful resource-oriented.** Standard CRUD verbs. RPC-style endpoints only where CRUD doesn't fit (scan trigger, credit operations, trust decisions).
2. **Money as `_minor` integers** in all request/response bodies per A.10 C1 / REQ-17.
3. **`user_edited_at` timestamps** returned on editable fields; sent on updates per A.10 C2 / REQ-13.
4. **Authentication:** Firebase ID token in `Authorization: Bearer <token>` header. Backend verifies via Firebase Admin SDK. All endpoints except health require auth.
5. **Ownership scoping:** Backend resolves `ownership_scope_id` from the authenticated user. Clients never send scope IDs.
6. **Pagination:** Cursor-based (`?cursor=<opaque>&limit=N`). Default limit 50, max 200.
7. **Real-time:** Firestore `onSnapshot` replaced by TanStack Query polling + SSE for scan events (REQ-04).
8. **Idempotency:** Mutation endpoints accept `Idempotency-Key` header per A.10 primitive #6.
9. **Error envelope:** `{ "error": { "code": "string", "message": "string", "details": {} } }` with appropriate HTTP status codes.

### GET `/scans/:scan_id/events` (SSE)

Server-Sent Events stream. Seven-state union per A.10 primitive #11 / A.9 SCAN-EVENTS contract (REQ-04).

```
GET /scans/{scan_id}/events
Accept: text/event-stream
Last-Event-Id: <uuid>              // for replay-on-reconnect
```

```
event: scan_progress
id: <event_uuid>
data: {"scan_id":"uuid","event_type":"queued","payload":null,"created_at":"..."}

event: scan_progress
id: <event_uuid>
data: {"scan_id":"uuid","event_type":"picked_up","payload":null,"created_at":"..."}

event: scan_progress
id: <event_uuid>
data: {"scan_id":"uuid","event_type":"llm_start","payload":{"model":"gemini-2.0-flash"},"created_at":"..."}

event: scan_progress
id: <event_uuid>
data: {"scan_id":"uuid","event_type":"llm_end","payload":{"token_count":1500},"created_at":"..."}

event: scan_progress
id: <event_uuid>
data: {"scan_id":"uuid","event_type":"reconciling","payload":null,"created_at":"..."}

event: scan_progress
id: <event_uuid>
data: {"scan_id":"uuid","event_type":"completed","payload":{"transaction_id":"uuid"},"created_at":"..."}
```

On `completed`: payload contains the created `transaction_id`. On `failed`: payload contains `error_code` + `error_message`.

Dual-transport binding (REQ-04): SSE for web clients, WebSocket for mobile. Both emit identical `scan_event` payloads from `scan_event_log`.

Recovery: `Last-Event-Id` header replays missed events from `scan_event_log` (A.10 primitive #11 daily-partitioned backing store).

## Migration Notes: Frontend Changes

When the frontend migrates from Firestore-direct to this API:

1. **`services/` and `repositories/` dissolve.** Replace with OpenAPI-generated client + TanStack Query hooks in `hooks/data/`.
2. **`onSnapshot` listeners â†’ TanStack Query.** Configure `staleTime`, `refetchInterval`, and `queryKey` invalidation per resource.
3. **Scan flow â†’ SSE.** Replace Cloud Function call + Firestore `onSnapshot` with POST `/scans` + EventSource at `/scans/:scan_id/events`.
4. **Money representation.** Frontend must convert `_minor` integers to display values using the currency's exponent from `/ref/currencies`.
5. **`user_edited_at` tracking.** PATCH requests for transactions must send only fields the user actually changed; backend sets `*_user_edited_at` automatically.
6. **Gemini calls.** No more client-side Gemini API key. All AI calls go through POST `/scans` (backend proxies to Gemini).
