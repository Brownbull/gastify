1 · Add a spacing scale foundation

The study's #1 finding. Gustify routes every gap/padding through named steps (pg-8, pg-10…); gastify has none, so components improvise (gap-2.5, p-4, px-3). One vocabulary is the main reason Gustify feels cohesive.

Reason: consistent spacing across every component, set once. Pure-additive (new gt-N utilities), no visual change on its own — it just makes every later change land consistently.

Tradeoff: a one-time pass converting components from raw Tailwind to gt-N. Low risk, some churn. Decision needed: keep off-grid 6/10px steps (recommended — matches Gustify) or grid-only 4/8/12.

4 · Smaller adopts (pick any)

  • Framed IconTile atom — one reusable icon-well (44/64px, border + tiny shadow) shared by ItemRow, EmptyState, notices. Falls out of the ItemRow work.
  • font-medium muted meta — drop the meta/sub line from bold to medium-muted so long copy reads. Tiny change, big legibility win.
  • StatusCard / Notice molecule — tinted surface + solid bordered icon disc + medium body, for over-budget / scan warnings. New molecule.
  • EmptyState polish — 64px framed icon tile + stacked→row actions on wider screens.
  • “Selection = depth” rule — unselected controls recede (soft border, no shadow); selected/hover gain ink border + tiny shadow. Codify across chips/tabs/options.
