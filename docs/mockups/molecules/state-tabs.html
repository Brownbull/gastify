State tabs

Canonical multi-state pattern. Toggles which `.state-content` is visible. Used by single-scan, login, consent, and any screen with mode/variant switching. tweaks.js binds click handlers; both legacy (.state-tabs > .state-tab[data-state]) and ARIA ([role="tablist"] > [role="tab"]) shapes work.

Variants — legacy shape (data-state)

3-state · idle / processing / error
Idle. Camera ready, awaiting tap.
Receipt frame · tap to capture · long-press for mode menu
Processing. AI extracting line items.
Spinner · "Leyendo recibo..." · cancel button
Error. Low confidence · retry or fall back to manual.
Error icon · "No pudimos leer el recibo" · Reintentar / Manual

Variants — ARIA shape ([role="tablist"])

2-state · normal / hard-lock
Normal mode. Free-form edits across every transaction field.
Hard-lock mode. Amount + date frozen; only category, tags, notes editable. Per REQ-12 / REQ-13.

Composition contract

  • Atoms used (one level down): none — primitive molecule. Composes raw button + text directly.
  • JS binding: tweaks.js owns the click → toggle. Author HTML only; no per-screen JS.
  • A11y: ARIA shape preferred for production (role="tablist" + role="tab" + aria-selected + aria-controls + role="tabpanel"). Legacy shape acceptable for prototypes.
  • Keyboard: arrow-key navigation expected — tweaks.js to land in M3 follow-up if not present (track in COMPONENT-LIBRARY.md state matrix).

Usage

<!-- ARIA shape (preferred) -->
<div class="state-tabs" role="tablist">
  <button class="state-tab" role="tab" aria-selected="true" aria-controls="p-idle">Idle</button>
  <button class="state-tab" role="tab" aria-selected="false" aria-controls="p-error">Error</button>
</div>
<div class="state-content is-active" role="tabpanel" id="p-idle">...</div>
<div class="state-content"            role="tabpanel" id="p-error" hidden>...</div>

<!-- Legacy shape (prototypes) -->
<div class="state-tabs">
  <button class="state-tab is-active" data-state="a">A</button>
  <button class="state-tab"           data-state="b">B</button>
</div>
<div class="state-content is-active" data-state="a">...</div>
<div class="state-content"            data-state="b">...</div>
