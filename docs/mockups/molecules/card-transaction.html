Card · Transaction

Pixel-faithful port of the legacy .txn-card. 40×50 receipt thumbnail · 28px borderless category badge with solid --cat-X background. Icon style is toggleable via Tweaks panel → Icons: pixel (default — legacy l1-{name}.svg, rendered white via filter: invert(1)) or emoji (Unicode glyphs from data-cat-emoji attribute, native multi-color OS rendering — matches gastify-history.html legacy mockup). 13px uppercase merchant tinted with --cat-X · 11px pills · 14px amount · 14px chevron with 6px gap below. Default sign is negative (no prefix); income variant adds leading + in --success.

States

Composition

  • Atoms used (one level down): skeleton (loading state)
  • Used by screens: Dashboard recent-tx, History, Group transactions, Insights
  • Variants: with-pills (default — date · location · item-count chips, matches legacy gastify-history) · without-pills (inline meta) · income/refund (leading +, --success) · expanded (items list with prices + "+N más" + "Ver transacción completa")
  • Category coupling. Set data-cat="X" on the <article> AND apply .is-X to the badge. The merchant text inherits var(--cat-X) automatically; the badge background matches. Both stay in sync per theme via the shell --cat-* tokens.
  • Categories supported (gastify shell): supermercado · alimentacion · transporte · farmacia · mascotas · vet · hogar · construccion · entretenimiento · servicios · vestuario · finanzas · salud · otros · refund (positive). 12 L1 SVG icons live at assets/icons/categories/l1/l1-{name}.svg.
  • Sign convention: default = negative, no minus prefix · positive needs .is-positive on the amount span (auto-prepends +, uses --success). Negative amounts stay --ink (black) — never red.
  • Layout (legacy faithful): 40px thumb | 1fr body | auto right · 10/14 padding · 8 gap. Thumbnail 40×50 with 4 fixed-pixel-width inner lines centered + opacity 0.5. Badge 28×28 borderless. Merchant 13px uppercase weight 700. Pills 11px weight 500 with 3/9 padding and 3 gap. Amount 14px weight 700. Chevron 14px opacity 0.4 with 6px gap below amount.
  • Badge icon (toggleable): badge background is always solid var(--cat-X), no border. The icon swaps based on body[data-icon-style] driven by Tweaks → Icons.
    • pixel (default): <img class="cat-icon-pixel"> loads legacy l1-{name}.svg rendered white via filter: invert(1) (the redundant brightness(0) is gone — that was painting the SVG's hidden 4% background rect white).
    • emoji: ::before pseudo-element pulls the glyph from data-cat-emoji on the badge — zero extra DOM, OS draws the multi-color emoji.
    Both branches always share the same --cat-X background, so the toggle changes only the foreground.
  • State matrix: default · hover · expanded (aria-expanded="true") · selected · disabled · loading · static (no chevron, no toggle)
  • A11y: outer article carries aria-expanded; header is a <button> that toggles the controlled detail id. SVG icons via <img alt=""> are decorative — the merchant text + amount carry the semantic.
