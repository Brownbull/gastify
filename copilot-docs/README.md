# copilot-docs

Project documentation generated by the
[project-documenter](https://github.com/github/awesome-copilot/blob/main/plugins/project-documenter/README.md)
plugin from [Awesome Copilot](https://github.com/github/awesome-copilot).

## Contents

| File | Description |
|------|-------------|
| [`project-summary.md`](project-summary.md) | 10-section project summary (executive summary, architecture, pipelines, components, API contracts, infrastructure, extension patterns, rules, dependencies, code structure) |
| [`diagrams/high-level-architecture.drawio`](diagrams/high-level-architecture.drawio) | C4 Context — system in its environment (clients, backend, external services, Railway) |
| [`diagrams/processing-pipeline.drawio`](diagrams/processing-pipeline.drawio) | C4 Container — receipt scan pipeline + statement reconciliation pipeline |
| [`diagrams/component-relationships.drawio`](diagrams/component-relationships.drawio) | C4 Component — gravity wells G1–G7 with interfaces and cross-well dependencies |

## Viewing / editing diagrams

Open any `.drawio` file with:

- **[draw.io desktop](https://github.com/jgraph/drawio-desktop/releases)** — double-click the file.
- **[app.diagrams.net](https://app.diagrams.net)** — File → Open from → Device.
- **VS Code** — install the
  [Draw.io Integration](https://marketplace.visualstudio.com/items?itemName=hediet.vscode-drawio)
  extension, then open the `.drawio` file directly.

## Regenerating

Re-run the plugin at any time to refresh all artifacts:

```bash
copilot plugin install project-documenter@awesome-copilot
# then in a Copilot session:
@project-documenter document this project, output to copilot-docs/
```
