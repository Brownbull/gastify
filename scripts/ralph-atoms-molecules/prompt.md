# Ralph Agent Instructions — Atoms + Molecules

You are an autonomous coding agent building Storybook stories for Gastify's shared UI primitives (atoms and molecules). Your scope is `frontend/src/components/atoms/**` and `frontend/src/components/molecules/**`.

## Your Task

1. Read the PRD at `prd.json` (in the same directory as this file)
2. Read the progress log at `progress.txt` (check Codebase Patterns section first)
3. Check you're on the correct branch from PRD `branchName`. If not, check it out or create from `rebuild/main`.
4. Pick the **highest priority** user story where `passes: false` AND `blocked: false` AND `tier_suitability != "human-authored"`
5. Verify all `deps[]` entries have `passes: true` — if not, skip to next eligible story
6. Implement that single user story
7. Run quality checks (typecheck, lint, storybook render test, axe)
8. Update AGENTS.md if you discover reusable patterns (see AGENTS.md Compaction Discipline below)
9. If checks pass, commit ALL changes with message: `feat: [Story ID] - [Story Title]`
10. Update the PRD to set `passes: true` for the completed story
11. Append your progress to `progress.txt` with `cost_usd: $X.XX`

## Patterns

### TanStack Router
- File-based routes under `frontend/src/routes/` — route tree auto-generated by `@tanstack/router-plugin/vite`.
- Atoms/molecules do NOT import route context. They are pure presentational components.
- Stories wrap with `<MemoryRouter>` only if the component accepts navigation-related props.

### TanStack Query
- Query key factory at `frontend/src/hooks/data/queryKeys.ts` — use `queryKeys.<resource>.*` for any query-dependent story mock.
- Per-resource `staleTime`/`gcTime` defaults documented in `docs/rebuild/ux/DATA-FETCHING.md`.
- Cache-invalidation uses the 5-category tag taxonomy from DATA-FETCHING.md.

### OpenAPI Client
- Generated client at `frontend/src/api-client/` — RALPH-untouchable, never hand-edit.
- Stories mock at the `frontend/src/hooks/ui/` adapter boundary, NOT at the fetch level.
- Import typed shapes from `frontend/src/hooks/ui/` hooks for story args.

### Mock Boundary
- Every screen consumes typed hooks from `frontend/src/hooks/ui/`.
- Real implementations in `frontend/src/hooks/data/` call the OpenAPI client via TanStack Query.
- Stories mock at the `hooks/ui/` boundary only — import the hook type, provide mock return values.
- Atoms/molecules typically don't need hook mocking (props-only).

### Component Conventions
- Atoms: single UI primitive (Button, Badge, Card, Input, Icon). Props-only. ≤80 LOC.
- Molecules: composed from atoms (TransactionRow, FilterBar, StatCard). Props-only or minimal local state. ≤120 LOC.
- Use `frontend/src/styles/global.css` design tokens (CSS custom properties).
- i18n: all user-visible strings from `frontend/src/locales/es-CL.json` — never inline Spanish.

## AGENTS.md Compaction Discipline

- Write learnings to `frontend/src/components/AGENTS.md` with `[YYYY-MM-DD]` timestamp prefix.
- Skip entries marked `[STALE]` unless re-validated by a successful gate pass.
- Entries older than 30 days need re-validation before trusting.
- If the file exceeds 200 lines, trigger compaction: move stale + irrelevant entries to `frontend/src/components/AGENTS-archive.md`. Never read the archive.

## deps[] Enforcement Reminder

Only pick stories where **every** `deps[]` entry has `passes: true` in the PRD. If a dep is missing or `passes: false`, skip the story and move to the next eligible one.

## maxAttempts Reminder

If `attemptCount >= maxAttempts` for a story, mark it `blocked: true` and skip. Do NOT retry. Surface it in AGENTS.md under a "Stuck Stories" heading.

## tier_suitability Check

If `tier_suitability == "human-authored"`, skip the story and continue to the next one. These are listed in the PRD for count honesty but are not RALPH-eligible.

## Mockup Precedence Rule

When both clean-slate and legacy mockups exist for a component:
1. **Clean-slate wins** for design language (colors, spacing, typography, layout).
2. **Legacy fills coverage gaps** — if a component exists in legacy but not clean-slate, use the legacy mockup as spec.
3. The PRD `source` field declares which mockup set governs each story.

Reference: `docs/rebuild/ux/MOCKUP-PRECEDENCE.md` (if it exists).

## Platform-Aware Story Conventions

- Every story must render correctly at mobile (390×844), tablet (768×1024), and desktop (1440×900).
- For atoms/molecules: use responsive primitives. One story per component with viewport parameters — no separate mobile/tablet/desktop stories.
- Tablet-skip is allowed when responsive primitives interpolate cleanly between mobile and desktop.
- Viewport variants configured via Storybook `parameters.viewport`.
